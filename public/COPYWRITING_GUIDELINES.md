# Prism Analytics - Copywriting Guidelines

**Quick Reference for Maintaining Brand Voice and Messaging Consistency**

---

## Brand Voice Quick Reference

### ✓ DO Write Like This
```
"Your entire product team gets unlimited access for one price."
"Answers in seconds. Not days. Not weeks."
"We're 40-50% cheaper than comparable platforms while being 3x faster."
"Save 150+ hours per quarter on analytics work."
```

### ✗ DON'T Write Like This
```
"Leveraging our synergistic analytics platform enables optimal stakeholder alignment."
"We're the industry-leading solution for enhanced data visualization paradigms."
"Cost savings are maximized through our proprietary event-streaming architecture."
```

## Messaging Hierarchy

### Level 1: Core Value (Top of Page)
**Primary:** Speed of insight + accessibility
```
"Get answers to 'why' in seconds. Not days. Not weeks."
"Non-technical PMs can build dashboards without tickets to data team."
```

### Level 2: Problem Validation (Mid-Page)
**Purpose:** Show you understand their pain
```
"Your data team is buried. Simple product questions turn into 2-week projects."
"By the time you get the numbers, the moment to act has passed."
```

### Level 3: Feature Depth (Features Section)
**Purpose:** Build confidence in capability
```
"Visual funnel builder—no SQL required."
"Real-time alerts on any metric."
"AI-powered anomaly detection finds patterns humans miss."
```

### Level 4: Proof (Social Proof & Pricing)
**Purpose:** Make the ask easy
```
"3x faster shipping" (peer validation)
"Average ROI payback: 6 weeks" (financial justification)
"14-day free trial, no credit card" (low friction)
```

## Sentence Structure for Clarity

### Rule 1: Lead with Benefit, Then Explain
❌ "Prism uses proprietary event-streaming architecture with real-time ingestion."
✓ "See dashboards update live as users interact with your product."

❌ "Our platform implements AI-driven anomaly detection algorithms."
✓ "Catch problems before customers do with automated alerts."

### Rule 2: Specific > Vague
❌ "We're much faster"
✓ "Answers in seconds instead of days"

❌ "Saves a lot of time"
✓ "150+ hours per quarter"

❌ "Many companies use us"
✓ "200+ Series A–C teams"

### Rule 3: "The Why" Before "The What"
❌ "Real-time event ingestion with sub-second latency."
✓ "Never miss a spike or anomaly. Catch problems before customers do."

❌ "API access for all Growth tier customers."
✓ "Export any data, build custom integrations, own your analytics."

### Rule 4: Active Voice (Usually)
❌ "Friction points are identified in seconds."
✓ "Identify friction points in seconds."

❌ "Was used by $45M Series B company."
✓ "One VP Product at a $45M Series B firm cut analysis time from days to hours."

## Word Choice Guide

### Preferred Language
| Instead of | Use |
|------------|-----|
| "Enables" | "Helps," "lets," "gets you" |
| "Leverage" | "Use," "tap into" |
| "Synergistic" | "Built together," "works seamlessly" |
| "Paradigm" | "Way," "approach" |
| "Utilize" | "Use" |
| "Comprehensive" | "Complete," "all in one" |
| "Scalable" | "Grows with you," "scales from [X] to [Y]" |
| "Innovative" | Show, don't tell—describe the actual feature |
| "Cutting-edge" | Show, don't tell—describe the actual benefit |
| "Next-generation" | Show, don't tell—compare to old way |

### Numbers Are Your Friend
- Always be specific
- Include the comparison (vs what?)
- Include the timeframe (per year? per quarter?)
- Include the audience (for Series A teams? all company sizes?)

```
❌ "Significant time savings"
✓ "150+ hours saved per quarter for Series B/C teams"

❌ "More affordable"
✓ "40-50% cheaper than Amplitude or Mixpanel"

❌ "Faster setup"
✓ "Most teams up and running in 30 minutes"
```

## Testimonial Review Checklist

When collecting or writing testimonials:

- [ ] **Specific metric included** ("3x faster" not "much faster")
- [ ] **Real job title included** (VP Product, not "Head of Analytics")
- [ ] **Real company size included** ("$45M Series B" adds credibility)
- [ ] **Benefit-focused** (what changed, not how the tool works)
- [ ] **Authentic tone** (sounds like a real person, not marketing copy)
- [ ] **Short enough** (2-3 sentences max)
- [ ] **Unique perspective** (don't repeat same message in every testimonial)

### Good Testimonial
"Prism cut our product analysis cycle from days to hours. Our team ships 3x faster now." — Sarah Chen, VP Product at Growth SaaS Co. ($45M Series B)

### Bad Testimonial
"Prism Analytics is an innovative, cutting-edge solution that leverages real-time insights to empower data-driven decision-making." — Anonymous

## FAQ Writing Principles

### Rule 1: Answer the Actual Question
❌ Q: "How long does setup take?" A: "We've streamlined our onboarding process to be as efficient as possible."
✓ Q: "How long does setup take?" A: "Most teams are up and running in under 30 minutes..."

### Rule 2: Address Unstated Objections
Not just "yes/no" — answer why this matters.
```
Q: "Can I connect Prism to my existing data warehouse?"
A: "Yes. We can connect to BigQuery, Snowflake, Redshift, Postgres, and others. 
We'll read from your existing data without copying anything. This means your data 
stays in your control, and we respect your existing security and compliance setup."
```

### Rule 3: Go Longer Than You Think
FAQs are low-friction. People are more likely to read them than long-form copy.
Aim for 2-3 paragraph answers.

### Rule 4: Include Reassurance
Not just capability, but comfort.
```
❌ "You can update your subscription anytime."
✓ "You can upgrade mid-month or downgrade at any time. No long-term lock-in. 
We'll prorate your charges fairly."
```

## Pricing Copy Principles

### Be Transparent About Why You Charge What You Charge
```
"Event-Based Pricing (Not Per-Seat): Your entire product team gets unlimited access 
for one price. Invite everyone—engineers, designers, PMs, leadership. We don't charge 
per user because insights should be accessible to everyone who needs them."
```

### Show the ROI Math
```
"Our customers typically save 150+ hours per quarter on analytics work. That's 
$30k-$50k of data team time freed up per year. Our pricing starts at $499/month 
because we're 40-50% cheaper than comparable platforms while being 3x faster."
```

### Make Annual Discount Obvious But Not Aggressive
```
"Annual Commitment = 25% Discount. Lock in lower rates and we'll give you extra 
support. Growth plan becomes $13,491/year (was $17,988)."
```

### Validate Tier Positioning
```
- Starter: "Best for early-stage teams validating product-market fit"
- Growth: "Most popular for Series B/C teams"
- Enterprise: "Custom enterprise pricing"
```

## Objection-Handling Language

### When Addressing Concerns, Use This Pattern:

1. **Acknowledge** the concern honestly
2. **Explain** why we built it this way
3. **Provide** reassurance with proof/example

```
Concern: "We already use Amplitude. Do we need Prism?"

Acknowledge: "Many of our best customers started with Amplitude or Mixpanel."
Explain: "They realized these tools weren't designed for how product teams actually 
work. Prism is 3x faster for common product analyses and doesn't require SQL expertise."
Reassure: "You can keep your existing setup or migrate gradually. No rush. One customer 
set it up alongside Amplitude for 3 months before switching completely."
```

## Section-Specific Writing Tips

### HERO SECTION
- Lead with pain relief, not features
- Make the next step obvious (free trial, no card)
- Include one social proof element (customer count, user count)

### PROBLEM SECTION
- Use their language (not "analytics fragmentation," but "analytics live in 5 tools")
- Validation: "Every single team had this problem"
- Never make them feel stupid for having this problem

### SOLUTION SECTION
- Direct contrast to each problem
- Include the word "but" or "instead" to show difference
- "We built this specifically for..."

### FEATURES SECTION
- Lead with benefit, not capability
- Include one real use case per feature
- Show, don't tell—provide examples

### TESTIMONIALS
- Use real quotes when possible
- Include specifics: title, company, metric
- Variety: different company sizes, different personas

### PRICING SECTION
- Justify the number (why this price?)
- Show ROI math clearly
- Address the unspoken question: "Is this worth it?"

### FAQ SECTION
- Answer more than asked
- Preemptively address concerns
- Be helpful, not defensive

## A/B Testing Framework

### Always Test Against Current Version
```
Hypothesis: [We believe that] [changing X] [will increase Y by Z%]

Current: [Include current copy]
Variant: [Include test copy]
Metric: [CTA click rate / signup rate / etc]
Duration: [2 weeks minimum]
Winner: [Note the result]
```

### Examples of Good Tests
- Hero headline variations (5+ variants, parallel test)
- Primary benefit emphasis (speed vs accessibility vs ROI)
- CTA text ("Start Free Trial" vs "Try Prism")
- Social proof position (above or below fold)
- Testimonial order (biggest company first vs specific benefit first)
- FAQ ordering (most clicked questions first)

### Bad Tests to Avoid
- Multiple changes at once (can't isolate what worked)
- Running 1 week (not enough statistical power)
- Testing design instead of copy (change one at a time)

## Voice Test: Is This Copy Ready?

Read it out loud. If it sounds like how your actual customers talk to each other, it's good. If it sounds like marketing copy, rewrite it.

```
❌ "Prism Analytics represents a paradigm shift in product analytics methodology."
(Sounds like marketing)

✓ "Your product team finally has the insights you need to make decisions with confidence."
(Sounds like conversation)
```
